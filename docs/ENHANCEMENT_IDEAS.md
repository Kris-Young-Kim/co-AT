## 1. AI 기능 고도화

### 1.1 RAG 챗봇 구현 (규정 검색 AI)

**현재 상태**: TODO에 명시되어 있으나 미구현  
**구현 난이도**: ⭐⭐ (쉬움) - 기존 Gemini 인프라 활용 가능

**제안 내용**:

#### 데이터 소스

- **보조기기센터사업안내.md**: 정부 지원사업 안내, 운영 지침 포함
  - 5대 사업 상세 가이드 (상담, 체험, 맞춤형, 사후관리, 교육홍보)
  - 예산 집행 기준, 인력 운영 기준
  - 평가 지표 및 보고서 양식
  - 첨부 서식 28개 (상담 기록지, 평가지, 진행 기록지 등)

#### 구현 방법 (간단한 접근)

**옵션 1: Gemini Embedding + 간단한 검색 (추천)**

- **문서 청크 분할**: `보조기기센터사업안내.md`를 의미 단위로 분할
- **Gemini Embedding API**: 각 청크를 벡터로 변환
- **Supabase 저장**: `regulations` 테이블에 청크 + 벡터 저장
- **유사도 검색**: 질문을 벡터화하여 유사한 청크 검색
- **Gemini로 답변 생성**: 검색된 청크를 컨텍스트로 제공하여 답변 생성

**옵션 2: Supabase pgvector 활용 (고급)**

- **pgvector 확장 활성화**: Supabase에서 벡터 확장 설치
- **벡터 저장**: `regulations` 테이블에 `embedding vector(768)` 컬럼 추가
- **유사도 검색 쿼리**: PostgreSQL의 `<=>` 연산자로 코사인 유사도 계산
- **성능 최적화**: 인덱스 추가로 검색 속도 향상

#### 구현 단계

1. **데이터 준비** (1일)

   - `docs/보조기기센터사업안내.md` 파일 읽기
   - 문서를 섹션별로 분할 (예: "대여 기준", "수리비 한도", "평가지표" 등)
   - 각 섹션을 청크로 분할 (500-1000자 단위)

2. **벡터화 및 저장** (1일)

   - `actions/rag-actions.ts` 작성
   - `embedRegulations()` 함수: 문서 청크를 Gemini Embedding으로 변환
   - Supabase `regulations` 테이블에 저장
   - 마이그레이션 파일 생성: `migrations/012_create_regulations.sql`

3. **검색 및 답변 생성** (2일)

   - `searchRegulations(query: string)` 함수: 질문을 벡터화하여 유사 청크 검색
   - `generateRegulationAnswer(query: string, context: string[])` 함수: Gemini로 답변 생성
   - System Prompt에 "보조기기센터 운영 지침서 기반으로 답변" 명시

4. **UI 컴포넌트** (2일)
   - `components/features/chat/RegulationChatbot.tsx` 작성
   - 채팅 인터페이스 (질문 입력, 답변 표시)
   - 로딩 상태, 에러 처리
   - 관리자 대시보드에 챗봇 위젯 추가

#### 예상 질문 예시

- "대여 기간은 얼마나 되나요?"
- "수리비 한도는 얼마인가요?"
- "맞춤제작 지원금은 얼마인가요?"
- "평가지표는 어떻게 구성되어 있나요?"
- "인력 운영 기준은 무엇인가요?"

#### 기술 스택

- **Gemini 1.5 Flash**: 기존 인프라 활용 (이미 `lib/gemini/client.ts` 존재)
- **Supabase**: 벡터 저장 및 검색
- **Next.js Server Actions**: `actions/rag-actions.ts`

**기대 효과**:

- 규정 검색 시간 90% 단축
- 일관된 업무 처리 기준 제공
- 신규 직원 온보딩 시간 단축
- 정부 지원사업 안내 자동화

**구현 우선순위**: ⭐⭐⭐ (높음)  
**예상 소요 시간**: 1주 (5일)
