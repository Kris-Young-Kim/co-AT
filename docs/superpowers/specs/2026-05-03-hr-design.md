# GWATC HR 앱 설계 (Phase 5)

**작성일:** 2026-05-03
**앱:** `apps/hr` → `hr.gwatc.cloud`
**대상 사용자:** ADMIN, MANAGER, STAFF

---

## 1. 목표

인사관리, 급여관리, 일용급여 관리, 근태관리, 경력관리, 호봉관리, 증명서 발급 기능을 하나의 HR 앱으로 통합한다.
향후 기능 추가를 고려해 모듈별로 명확히 분리된 구조로 설계한다.

---

## 2. 단계별 출시 계획

| 단계 | 기능 | 전제 조건 |
|---|---|---|
| **HR-1** | 인사관리, 근태관리, 경력관리 | 없음 (기초 데이터) |
| **HR-2** | 호봉관리, 급여관리, 일용급여 관리 | HR-1 완료 |
| **HR-3** | 증명서 발급 | HR-1, HR-2 완료 |

---

## 3. 데이터 모델

모든 테이블은 `hr_` 네임스페이스를 사용하며 RLS를 활성화한다.

### HR-1 테이블

#### `hr_employees` — 직원 프로필

```sql
id             uuid PRIMARY KEY DEFAULT gen_random_uuid()
clerk_user_id  text UNIQUE       -- Clerk 연동 직원은 값 있음, 비연동은 NULL
name           text NOT NULL
email          text NOT NULL
phone          text
department     text NOT NULL     -- 예: '보조공학팀', '행정팀'
position       text NOT NULL     -- 예: '보조공학사', '팀장'
employment_type text NOT NULL    -- full_time | part_time | contract | daily
hire_date      date NOT NULL
leave_date     date              -- 퇴직일 (NULL = 재직 중)
is_active      boolean NOT NULL DEFAULT true
created_at     timestamptz NOT NULL DEFAULT now()
updated_at     timestamptz NOT NULL DEFAULT now()
```

#### `hr_careers` — 경력 관리

```sql
id             uuid PRIMARY KEY DEFAULT gen_random_uuid()
employee_id    uuid NOT NULL REFERENCES hr_employees(id) ON DELETE CASCADE
organization   text NOT NULL    -- 소속 기관/회사
position       text NOT NULL    -- 직책/직위
start_date     date NOT NULL
end_date       date             -- NULL = 현재 재직 중
description    text
created_at     timestamptz NOT NULL DEFAULT now()
```

#### `hr_attendance_records` — 출퇴근 기록

```sql
id             uuid PRIMARY KEY DEFAULT gen_random_uuid()
employee_id    uuid NOT NULL REFERENCES hr_employees(id) ON DELETE CASCADE
date           date NOT NULL
check_in       timestamptz
check_out      timestamptz
note           text
created_at     timestamptz NOT NULL DEFAULT now()
UNIQUE (employee_id, date)
```

#### `hr_leave_requests` — 연차·휴가 신청

```sql
id             uuid PRIMARY KEY DEFAULT gen_random_uuid()
employee_id    uuid NOT NULL REFERENCES hr_employees(id) ON DELETE CASCADE
leave_type     text NOT NULL    -- annual | sick | special | unpaid
start_date     date NOT NULL
end_date       date NOT NULL
days_used      numeric(4,1) NOT NULL   -- 0.5일 단위
reason         text
status         text NOT NULL DEFAULT 'pending'  -- pending | approved | rejected
reviewed_by    uuid REFERENCES hr_employees(id)
reviewed_at    timestamptz
created_at     timestamptz NOT NULL DEFAULT now()
```

### HR-2 테이블

#### `hr_salary_grades` — 호봉표 (센터 자체 정의)

```sql
id             uuid PRIMARY KEY DEFAULT gen_random_uuid()
grade_name     text NOT NULL UNIQUE  -- 예: '1급 1호봉', '2급 3호봉'
base_salary    integer NOT NULL      -- 기본급 (원)
is_active      boolean NOT NULL DEFAULT true
created_at     timestamptz NOT NULL DEFAULT now()
updated_at     timestamptz NOT NULL DEFAULT now()
```

#### `hr_allowance_types` — 수당 유형 (관리자 커스텀)

```sql
id             uuid PRIMARY KEY DEFAULT gen_random_uuid()
name           text NOT NULL UNIQUE  -- 식대, 위험수당, 직책수당, 명절수당, 커스텀
is_active      boolean NOT NULL DEFAULT true
created_at     timestamptz NOT NULL DEFAULT now()
```

기본 시드: 식대, 위험수당, 직책수당, 명절수당

#### `hr_salary_records` — 월별 급여 대장

```sql
id             uuid PRIMARY KEY DEFAULT gen_random_uuid()
employee_id    uuid NOT NULL REFERENCES hr_employees(id) ON DELETE CASCADE
year_month     char(7) NOT NULL   -- 'YYYY-MM' 형식 (예: '2026-05')
salary_grade_id uuid REFERENCES hr_salary_grades(id)
base_salary    integer NOT NULL
allowances     jsonb NOT NULL DEFAULT '[]'
               -- [{type_id, name, amount}]
deductions     jsonb NOT NULL DEFAULT '{}'
               -- {national_pension, health_insurance, employment_insurance, income_tax, local_tax}
gross_pay      integer NOT NULL   -- 지급 총액
net_pay        integer NOT NULL   -- 실수령액
note           text
confirmed_at   timestamptz        -- NULL = 미확정, 값 있음 = 확정
created_at     timestamptz NOT NULL DEFAULT now()
updated_at     timestamptz NOT NULL DEFAULT now()
UNIQUE (employee_id, year_month)
```

#### `hr_daily_wages` — 일용직 급여

```sql
id             uuid PRIMARY KEY DEFAULT gen_random_uuid()
employee_id    uuid NOT NULL REFERENCES hr_employees(id) ON DELETE CASCADE
work_date      date NOT NULL
hours_worked   numeric(4,1) NOT NULL
hourly_rate    integer NOT NULL
gross_pay      integer NOT NULL
deductions     jsonb NOT NULL DEFAULT '{}'
net_pay        integer NOT NULL
note           text
created_at     timestamptz NOT NULL DEFAULT now()
```

### HR-3 테이블

#### `hr_certificates` — 증명서 발급 이력

```sql
id             uuid PRIMARY KEY DEFAULT gen_random_uuid()
employee_id    uuid NOT NULL REFERENCES hr_employees(id) ON DELETE CASCADE
type           text NOT NULL   -- employment | career | salary | resignation
purpose        text            -- 사용 목적 (예: '금융기관 제출')
issued_by      uuid NOT NULL REFERENCES hr_employees(id)
               -- 발급자도 hr_employees에 등록되어 있어야 함 (ADMIN/MANAGER 포함)
issued_at      timestamptz NOT NULL DEFAULT now()
```

```
│   ├── leave-actions.ts            -- 휴가 신청/승인/반려
│   ├── career-actions.ts           -- 경력 CRUD
│   ├── salary-actions.ts           -- 급여 대장 생성/수정/확정
│   ├── salary-grade-actions.ts     -- 호봉표 CRUD
│   ├── allowance-type-actions.ts   -- 수당 유형 CRUD
│   ├── daily-wage-actions.ts       -- 일용급여 CRUD
│   └── certificate-actions.ts      -- 증명서 발급 + 이력 저장
└── lib/
    ├── permissions.ts              -- 역할 기반 접근 제어 헬퍼
    └── salary-calculator.ts        -- 4대보험/세금 공제 계산 로직
```

### 핵심 데이터 흐름

**직원 등록 (ADMIN)**
```
EmployeeForm → employee-actions.createEmployee
  → INSERT hr_employees
  → (clerk_user_id 있을 때) Clerk publicMetadata.apps[] 업데이트
```

**출퇴근 기록 (전 직원)**
```
AttendancePage → attendance-actions.upsertAttendance
  → UPSERT hr_attendance_records (employee_id + date UNIQUE)
  → RLS: 본인 INSERT, MANAGER/ADMIN 전체 조회
```

**휴가 신청 → 승인**
```
STAFF/MANAGER/ADMIN: LeaveForm → leave-actions.createLeaveRequest → status: pending
MANAGER/ADMIN: LeaveList → leave-actions.reviewLeave → status: approved | rejected
  → reviewed_by, reviewed_at 업데이트
```

**급여 계산**
```
SalaryPage → salary-actions.createSalaryRecord
  → salary-calculator.ts로 4대보험·세금 자동 계산
  → INSERT hr_salary_records (gross_pay, net_pay, deductions jsonb)
  → 확정: confirmed_at 업데이트 → 수정 잠금
```

**증명서 발급**
```
CertificatePage → certificate-actions.issueCertificate
  → INSERT hr_certificates (이력 저장)
  → 브라우저 print() 또는 PDF 다운로드
```

## 8. 연차 잔여일 계산 로직

- `hire_date` 기준 근속 연수로 연간 부여 일수 결정 (기본 15일, 3년 이상 +1일/년, 최대 25일)
- 당해 연도 `status = 'approved'` leave_requests의 `days_used` 합산 차감
- 대시보드 및 직원 상세에서 잔여 연차 표시

## 9. 급여 공제 계산 기준 (salary-calculator.ts)

| 항목 | 계산 기준 |
|---|---|
| 국민연금 | 기본급 × 4.5% |
| 건강보험 | 기본급 × 3.545% |
| 고용보험 | 기본급 × 0.9% |
| 산재보험 | 사업주 부담 (직원 공제 없음) |
| 소득세 | 간이세액표 적용 (근로소득 간이세액표) |
| 지방소득세 | 소득세 × 10% |

> 실제 운영 시 세율은 매년 고시 기준으로 관리자가 업데이트 가능하도록 설계

## 11. 기술 스택

- **프레임워크:** Next.js 16 App Router, React 19, TypeScript strict
- **스타일:** Tailwind CSS + shadcn/ui (`@co-at/ui`)
- **인증:** Clerk (`@co-at/auth`)
- **DB:** Supabase (`@co-at/lib`)
- **Server Actions 우선:** API Route는 외부 연동 시에만
