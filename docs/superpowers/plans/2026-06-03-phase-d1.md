# Phase D-1 — HR 인사관리 기초

**날짜**: 2026-06-03

## 범위

### 1. DB Migration (067)
- `hr_departments` (id, name, code, description, is_active)
- `hr_positions` (id, name, code, level int, is_active)
- `hr_salary_steps` (id, step_number, step_name, base_amount, is_active)
- `hr_salary_step_history` (id, employee_id, step_id, effective_date, reason, created_by)
- `hr_employees` 컬럼 추가: `department_id FK`, `position_id FK`, `salary_step_id FK`
  - 기존 `department`/`position` text 컬럼 유지 (하위호환)

### 2. Types 추가 (packages/types/src/hr.types.ts)
- `HrDepartment`, `HrPosition`, `HrSalaryStep`, `HrSalaryStepHistory`
- Input types

### 3. Actions
- `department-actions.ts`
- `position-actions.ts`
- `salary-step-actions.ts` (등록/수정/삭제 + 호봉승급)

### 4. Sidebar 재구성 (AppSidebar.tsx)
카테고리 토글 구조:
- 대시보드 (독립)
- 인사관리 ▼: 부서등록, 직급등록, 호봉등록, 호봉승급, 인사정보, 인사정보현황, 증명서발급
- 급여관리 ▼: 급여관리, 일용급여 (D-2에서 확장)
- 근태관리 ▼: 출퇴근기록, 연차·휴가, 경력관리 (D-3에서 확장)

### 5. Pages
- `/departments` — 부서 CRUD
- `/positions` — 직급 CRUD
- `/salary-steps` — 호봉 CRUD
- `/salary-step-promotions` — 호봉승급 이력
- `/employees/status` — 인사정보현황 (부서별/고용형태별 집계)
