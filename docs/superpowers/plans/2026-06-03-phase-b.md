# Phase B 구현 계획

**날짜**: 2026-06-03  
**작성자**: Claude Sonnet 4.6  
**목적**: eval 만족도·교육이력 수집, inventory 소독·세척 전용 관리

---

## 배경

Phase A(stats 실적보고 자동화 + 평가지표 위젯)가 완료되었으나,
EvalScoreWidget의 **서비스효과성 5점(서비스 만족도)**은 데이터 없어 "—" 상태.
교육이력 도메인 뷰, inventory 소독·세척 전용 UI도 미비.

---

## 구성 요소

### B-1: eval — 서비스 만족도 수집

**목표**: 서비스 종결 시 만족도(1-5점)를 기록 → stats EvalScoreWidget에서 서비스효과성 5점 자동 계산

**DB 변경**
- Migration `066_add_satisfaction_score.sql`
  - `eval_service_records.satisfaction_score INTEGER CHECK (satisfaction_score BETWEEN 1 AND 5)`
  - `eval_service_records.satisfaction_comment TEXT` (선택 코멘트)

**코드 변경**
1. `apps/eval/components/eval/ServiceRecordForm.tsx`
   - `is_closed = true` 조건부로 만족도 입력 섹션 표시 (1~5점 라디오)
   - 비고 입력 optional
2. `actions/service-record-actions.ts`
   - `createServiceRecord` input에 `satisfaction_score`, `satisfaction_comment` 추가
3. `apps/stats/actions/stats-actions.ts`
   - `getSatisfactionSummary(year)` 추가 — 평균 만족도, 응답 수 반환
4. `apps/stats/components/stats/EvalScoreWidget.tsx`
   - 서비스효과성 행에 실제 만족도 점수 표시 (평균 4.0 → 4점, 4.5+ → 5점)

---

### B-2: eval — 이용자 교육 이력

**목표**: `is_education=true` 서비스 기록을 AT 도메인(WC/ADL/…)별로 집계·표시

**코드 변경**
1. `apps/eval/app/education/page.tsx` (신규)
   - 연도/월/도메인 필터
   - 테이블: 날짜, 이름, 서비스영역(도메인), 내용, 담당자
   - 하단 도메인별 집계 카드
2. `actions/education-actions.ts` (신규)
   - `getEducationRecords({ year, month, domain })` — `is_education=true` 필터
3. `apps/eval/components/layout/EvalSidebar.tsx`
   - '교육 이력' 메뉴 항목 추가

---

### B-3: inventory — 소독·세척 전용 페이지

**목표**: `type='cleaning'` 유지보수 기록 전용 뷰 + 기기별 마지막 세척일 + 일괄 기록

**코드 변경**
1. `apps/inventory/app/cleaning/page.tsx` (신규)
   - 탭 1: 기기별 마지막 세척일 목록 (오래된 순 정렬)
   - 탭 2: 세척 이력 로그 (type='cleaning' 필터)
   - 일괄 세척 등록 폼 (날짜 + 담당자 + 기기 다중 선택)
2. `actions/cleaning-actions.ts` (신규)
   - `getCleaningHistory()` — 기기별 last_cleaned 집계
   - `logBatchCleaning(deviceIds, date, technician, notes)` — 다중 기기 세척 등록
3. `apps/inventory/components/layout/InventorySidebar.tsx`
   - '소독·세척' 메뉴 항목 추가
